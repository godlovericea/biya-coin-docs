# Cursor 使用心得分享

---

## 一、基础准备（使用前提和习惯）

### 1. 保持 Cursor 版本最新
删除除了 Cursor 以外的代码编辑器，比如 PyCharm、Goland。

### 2. 以文档形式记录
Cursor 输出的关键信息需要以文档形式记录，因为 Cursor 上下文环境存储空间有限。

### 3. 代码关键地方注释清晰
便于 Cursor 理解以及后期 Cursor 维护。

### 4. 写方法前先做拆分
写一个具体的接口或者方法的时候，如果自己知道怎么写，可以先把方法按照步骤拆分；如果不知道怎么写，让 Cursor 给你拆分，然后在他基础上再修改。

### 5. 优先级：解决思路 > 代码实现
在完成需求的时候，需要花更多的时间在解决思路上。

### 6. 每次输入描述一件事情
在使用 Cursor 的时候，输入的文字尽可能描述一件事情，避免在一次对话中混杂多个不相关的问题。

---

## 二、新项目开发（从0到1）

### 7. 技术选型与架构
对于新开发的项目，首先我们需要让 Cursor 知道我们要做什么需求，项目有哪些内容，最终达到什么目的，整体框架的技术栈是什么，项目中涉及到的技术点有什么。

**使用方法：**
在使用第三方包的时候，让 Cursor 帮你把某一项技术做对比，形成决策树。根据决策选项的得分，可以选择某一项具体的技术或者第三方包。

### 8. 项目文件目录设计
在了解完具体的需求以及技术点之后，让 Cursor 生成一个完整的项目文件图。

**使用方法：**
如果你觉得不合理，或者有文件需要换位置，这个时候需要跟 Cursor 提出观点，让 Cursor 修改。

### 9. 项目实施排期
根据 Cursor 提示，让 Cursor 帮你做详细的排期计划。排期计划中会包含各项技术要点，有疑问的一定要问 Cursor，然后更新文档。

**使用方法：**
排期计划应该具体到每个文件或者每个接口，一定要形成文档，便于 Cursor 审查和修改。

## 三、日常开发维护（工作场景）

### 10. 阅读文档
让 Cursor 总结出关键点，有分步骤执行的，需要梳理出步骤，以文档的形式记录，便于执行。

**使用方法：**
在聊天地方把文档地址链接粘贴上来，如果没有文档地址或者 Cursor 无法打开的地址，可以把离线文档以文件的形式存放在 docs 目录下，让 Cursor 阅读。

### 11. 审查优化代码，解决性能或者安全问题
直接告诉 Cursor 某一个具体的方法需要优化。

**使用方法：**
右键选择资源视图的某一个文件，拷贝相对路径或者绝对路径，告诉他这个文件在哪里，哪一个具体的方法需要优化，优化点是什么类型，是性能优化，还是代码安全检查，还是需要补充逻辑，一定要明确优化点，给 Cursor 一个明确的指令。

### 12. 解决 BUG
代码运行报错，或者接口报错的时候，把具体的接口，具体的错误信息完整的粘贴到 Cursor 输入框。

**使用方法：**
如果错误信息过多，可以以文档的形式存储错误，让 Cursor 阅读，按照 Cursor 提示，打印日志或者添加断点，调试代码。

### 13. 多项目调用解决思路
把涉及到的项目以及文档，放在一个大文件夹下面，在大文件夹路径打开 Cursor。如下图所示：

```
BigFolder    <---------Open With Cursor
├── Project A
├── Project B
├── Project C
├── File A
├── File B
...
```

**使用方法：**
做项目时候，需要明确告知，你正在完成哪个项目（文件夹）的什么工作，需要调用哪个项目（文件夹）的哪一个方法，位置一定要明确。

### 14. 维护老旧代码思路
此时需要根据具体场景处置。

**使用方法：**
如果是在老代码基础上修改或者新增需求，可以让 Cursor 在老代码基础上新写一个方法，新的需求写在这个新的方法里。如果这个方法关联到其他很多方法，最好是把涉及到的方法，写入一个新的文件。

**核心原则**：按照最小修改范围原则，维持老代码不变，减少测试工作。

---

## 总结

使用 Cursor 的核心思路：
1. **文档化思维**：所有重要信息都要形成文档
2. **明确指令**：告诉 Cursor 明确的位置、明确的需求、明确的优化点
3. **思路优先**：先想清楚怎么做，再让 Cursor 帮你实现
4. **最小改动**：特别是维护老代码时，尽量减少修改范围
