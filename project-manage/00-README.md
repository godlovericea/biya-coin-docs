## 🚀 使用指南

### 新手上路（第1周）

**Day 1-2：建立认知**
```
1. 阅读《从开发者到项目管理者》前半部分（角色转变）
2. 理解开发者思维 vs 项目管理者思维
3. 了解核心能力矩阵
```

**Day 3-4：学习方法**
```
1. 学习工时评估方法（三点估算法）
2. 学习任务拆解（WBS、SMART原则）
3. 下载《快速参考手册》备查
```

**Day 5：准备启动**
```
1. 使用《工时评估模板》和《任务拆解模板》规划第一个项目
2. 识别前3大风险
3. 设置每日站会和周报机制
```

---

### 日常使用（日常工作）

**每天**：
- 早上查看《风险管理指南》-风险登记表
- 每日站会（参考《快速参考手册》）
- 遇到问题时查阅对应章节

**每周**：
- 周五下午使用《项目周报模板》撰写周报
- 更新风险状态
- 规划下周任务

**每2周**：
- 风险评审会议（参考《风险管理指南》）
- 复盘迭代经验教训

**项目结束**：
- 对比《工时评估模板》中的预估与实际
- 更新偏差系数，优化下次评估

---

### 进阶学习（持续提升）

**第1个月**：
- [ ] 通读所有文档
- [ ] 负责一个小型项目（2-3人）
- [ ] 练习工时评估和任务拆解

**第2-3个月**：
- [ ] 负责中型项目（3-5人）
- [ ] 建立自己的风险清单
- [ ] 积累历史数据

**第4-6个月**：
- [ ] 负责较大项目（5人以上）
- [ ] 总结自己的项目管理方法论
- [ ] 辅导其他新PM

## 💡 实战建议

### ✅ 推荐做法

1. **循序渐进**：不要一次性学完所有内容，先读核心指南，再按需深入
2. **边学边做**：理论+实践结合，立即应用到实际项目中
3. **建立自己的模板库**：基于提供的模板，结合团队实际情况调整
4. **记录数据**：积累工时评估、风险案例等历史数据
5. **定期复盘**：每个项目结束后总结经验教训

### ❌ 避免做法

1. **生搬硬套**：不考虑实际情况，完全照搬模板
2. **过度流程化**：小项目也用复杂流程，增加负担
3. **纸上谈兵**：只学不做，没有实践
4. **忽视团队反馈**：流程不适合团队却不调整
5. **完美主义**：第一次就想做到完美，导致行动瘫痪
