# 产品经理必看：如何写AI友好的PRD

> 让开发用AI快速实现你的需求

---

## 为什么要写AI友好的文档？

**传统方式**：
```
产品写需求 → 开发问东问西 → 反复沟通 → 开始开发
耗时：2-3天
```

**AI辅助开发**：
```
产品写清晰的需求 → 开发用AI直接生成代码 → 快速实现
耗时：半天
```

**关键**：文档写得越清楚，AI生成的代码越准确！

---

## 核心原则（3个要点）

### 1. 简洁但完整
- ❌ 长篇大论，写一堆废话
- ✅ 直接说重点，但不遗漏关键信息

### 2. 流程要详细
- ❌ "用户可以签到获得积分"（太模糊）
- ✅ 用流程图说明每一步（见后面示例）

### 3. 规则要具体
- ❌ "根据订单金额给积分"（AI不知道怎么算）
- ✅ "积分 = Math.floor(实付金额 × 0.05)"（清晰明确）

---

## 文档必备5个部分

### 第1部分：需求背景（1段话）

**模板**：
```
【为什么做】
- 现状问题1
- 现状问题2

【目标】
- 提升XX指标 XX%
- 降低XX指标 XX%

【用户】
- 谁在用这个功能
```

**示例**：
```
【为什么做】
- 用户注册后活跃度低，30天留存率仅35%

【目标】  
- 提升日活 30%
- 提升30天留存至 60%

【用户】
- 所有注册用户
```

### 第2部分：功能列表（列清楚）

**模板**：
```
功能1：XXX
- 子功能1.1：具体做什么
- 子功能1.2：具体做什么

功能2：XXX
- 子功能2.1：具体做什么
```

**示例**：
```
功能1：赚取积分
- 每日签到：首次打开APP，自动签到，奖励10积分
- 完成订单：订单完成后，奖励实付金额×5%的积分
- 评价商品：写评价≥10字，奖励20积分，每天最多5次

功能2：使用积分  
- 积分商城：500积分换5元券，1000积分换10元券
- 订单抵扣：100积分=1元，每单最多抵30%
```

### 第3部分：完整流程（最重要！）

**用文字流程图表示**，格式如下：

```
用户操作
    ↓
系统判断：条件是什么？
    ├─ 分支1 → 结果1
    ├─ 分支2 → 结果2  
    └─ 分支3 ↓
继续下一步
    ↓
显示结果
    ↓
完成
```

**完整示例**：

```
【流程：用户签到】

用户打开APP
    ↓
系统检测：今天是否已签到？
    ├─ 是 → 显示"今日已签到" → 结束
    └─ 否 ↓
检查连续签到天数
    ├─ 连续3天 → 发20积分（10+10奖励）
    ├─ 连续7天 → 发60积分（10+50奖励）
    ├─ 连续30天 → 发210积分（10+200奖励）
    └─ 其他 → 发10积分
        ↓
弹窗提示："签到成功！获得XX积分"
    ↓
更新积分余额
    ↓
记录到明细
    ↓
完成
```

### 第4部分：业务规则（一张表搞定）

**用表格整理所有规则**：

| 项目 | 规则 | 备注 |
|-----|------|------|
| 签到积分 | 10/次，1次/天 | 连续签到有奖励 |
| 订单积分 | 实付×5% | 向下取整 |
| 评价积分 | 20/次，5次/天 | 文字≥10字 |
| 积分有效期 | 365天 | 过期清零 |

**如果有计算公式，直接写**：
```
订单积分 = Math.floor(实付金额 × 0.05)
抵扣金额 = Math.floor(积分数 ÷ 100)
```

### 第5部分：页面原型（用ASCII画）

**不需要专业设计工具**，用字符画出来就行：

```
┌──────────────────────────┐
│ < 返回   积分中心   规则  │
├──────────────────────────┤
│                          │
│    【我的积分：1,250】    │
│    [查看明细]             │
│                          │
│   今日任务                │
│   ✓ 每日签到    +10积分  │
│   ○ 完成订单    +50积分  │
│                          │
└──────────────────────────┘
```

**重点标注**：
- 按钮用 `[按钮文字]`
- 已完成用 `✓`
- 未完成用 `○`
- 输入框用 `[________]`

## 常见错误示例

### ❌ 错误1：描述太模糊

```
用户可以通过完成任务获得积分，积分可以用来兑换商品。
```

**问题**：
- 什么任务？
- 怎么计算积分？
- 能兑换什么商品？
- 兑换流程是什么？

### ✅ 正确写法：

```
【任务类型】
1. 每日签到：10积分/次，1次/天
2. 完成订单：实付金额×5%，无限制
3. 评价商品：20积分/次，5次/天

【兑换商品】
- 5元优惠券：500积分
- 10元优惠券：1000积分
- 20元优惠券：2000积分

【兑换流程】
用户选择商品 → 确认兑换 → 检查积分是否足够 
→ 扣除积分 → 发放商品 → 完成
```

---

### ❌ 错误2：没有异常处理

```
用户点击兑换按钮，扣除积分，发放商品。
```

**问题**：
- 积分不足怎么办？
- 商品没库存怎么办？
- 兑换失败怎么提示？

### ✅ 正确写法：

```
【正常流程】
用户点击兑换 → 扣除积分 → 发放商品 → 提示"兑换成功"

【异常处理】
- 积分不足：提示"积分不足，还差XX积分"，不扣积分
- 无库存：提示"该商品已兑完"，不扣积分
- 系统异常：提示"兑换失败，请稍后重试"，不扣积分
```

---

### ❌ 错误3：数字不明确

```
用户评价商品可以获得积分，有每日上限。
```

**问题**：
- 多少积分？
- 上限是多少？
- 评价有要求吗？

### ✅ 正确写法：

```
【评价任务】
- 奖励：20积分/次
- 限制：5次/天
- 要求：文字评价≥10个字
- 特殊：7天内删除评价，扣回积分
```

## 快速检查清单

**写完文档后，问自己这些问题**：

### 业务逻辑
- [ ] 每个功能的触发条件写清楚了吗？
- [ ] 每个流程的分支都考虑到了吗？
- [ ] 所有的数字都明确了吗？（不能有"适当"、"若干"这种词）

### 异常情况  
- [ ] 失败场景怎么处理？
- [ ] 用户看到什么提示？
- [ ] 需要回滚数据吗？

### 页面交互
- [ ] 每个按钮点击后会怎样？
- [ ] 弹窗长什么样？
- [ ] 加载状态怎么显示？

### 规则计算
- [ ] 所有涉及计算的公式都写了吗？
- [ ] 向上取整还是向下取整？
- [ ] 保留几位小数？

## 实战示例对比

### 场景：订单退款后积分处理

#### ❌ 模糊版本

```
订单退款后，积分也要退还给用户。
```

#### ✅ 清晰版本

```
【退款积分处理】

触发条件：
- 订单状态变为"已退款"

计算公式：
退还积分 = Math.floor(退款金额 × (使用积分 ÷ 订单金额))

示例：
- 订单金额：100元
- 使用积分：500（抵扣5元）
- 实付：95元
- 退款金额：50元
- 退还积分 = 50 × (500 ÷ 100) = 250积分

流程：
订单退款成功
    ↓
计算应退还的积分
    ↓
积分退回用户账户
    ↓
记录积分明细（类型：退款退还）
    ↓
推送通知："已退还XX积分"
    ↓
完成

特殊情况：
- 如果订单没使用积分，则无需处理
- 如果部分退款，按比例退还
- 如果全额退款，全额退还积分
```

## 总结

**好的PRD = 简洁 + 完整 + 清晰**

**3个关键**：
1. 流程图要详细（用文字画出来，生成HTML可演示原型）
2. 规则要具体（数字、公式要明确）
3. 异常要考虑（失败怎么办）
