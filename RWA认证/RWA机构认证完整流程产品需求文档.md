# RWA机构认证完整流程产品需求文档

## 文档信息
- **文档版本**: v3.0
- **创建日期**: 2024-11-06
- **最后更新**: 2024-11-07
- **产品名称**: RWA机构认证系统
- **核心功能**: KYB认证 → 合规审核 → UBO认证 → 账户操作人认证 → 专业投资者认证 → 机构风险测评（六合一连续流程）

---

## 1. 产品概述

### 1.1 背景与目标
RWA机构认证是一个**连续的、六阶段的认证流程**，将KYB、合规审核、UBO、账户操作人认证、专业投资者认证、机构风险测评整合为一个统一的用户体验。用户从头到尾完成所有认证步骤后，才会收到"认证完成，等待审核"的最终提示。

### 1.2 核心设计理念
- **连续性**：六个认证环节无缝衔接，用户感知为一个完整流程
- **进度透明**：全局进度条实时显示整体完成度
- **体验统一**：统一的UI风格、交互模式和提示规范
- **状态同步**：各阶段数据和状态实时同步，支持中断恢复
- **安全登录**：通过账户操作人的人脸识别实现安全便捷的登录验证
- **风险适配**：通过风险测评确保产品适配性

### 1.3 认证流程总览
```
首页 → KYB认证（3步） → 合规审核（1步） → UBO认证（3步+KYC） → 账户操作人认证（2步+KYC） → 专业投资者认证（1步） → 机构风险测评（4步） → 最终确认 → 提交成功
```

## 2. 整体流程交互设计

### 2.1 全局进度管理

#### 2.1.1 整体进度条
**显示时机**：除首页外的所有页面顶部显示

**进度计算**（6个阶段）：
- KYB认证：0-16.7%
- 合规审核：16.7-33.3%
- UBO认证：33.3-50%
- 账户操作人认证：50-66.7%
- 专业投资者认证：66.7-83.3%
- 机构风险测评：83.3-100%

**交互规则**：
- 进度条实时更新，平滑过渡动画
- 显示当前阶段名称（如"第一步：KYB认证"）
- 显示具体进度文字（如"2/6 阶段"）
- 显示当前子步骤（如"基本信息"、"上传中 2/5"）

#### 2.1.2 阶段指示器
**展示形式**：6个横向排列的阶段标签

**阶段列表**：
1. KYB认证
2. 合规审核
3. UBO认证
4. 账户操作人
5. 专业投资者
6. 风险测评

**状态样式**：
- **进行中**：蓝色高亮 + 加粗文字
- **未开始**：灰色文字
- **已完成**：蓝色文字（不加粗）

**交互行为**：
- 不可点击（用户必须按顺序完成）
- 鼠标悬停显示阶段说明（Tooltip）

### 2.2 流程跳转规则

#### 2.2.1 前进规则
- 必须按顺序完成：KYB → 合规审核 → UBO → 账户操作人 → 专业投资者 → 风险测评 → 最终确认
- 每个阶段完成后自动跳转至下一阶段
- 不允许跳过任何阶段

#### 2.2.2 返回规则
- 每个页面都有"返回"按钮
- 返回到上一个操作页面（非上一阶段）
- 首页可以"返回首页"（退出流程）

#### 2.2.3 中断恢复
- 用户中途退出，下次进入时从中断位置继续
- 已填写数据自动保存（草稿）
- 进度条显示实际完成进度

### 2.3 数据传递机制

**阶段间数据共享**：
- KYB阶段采集的公司信息自动传递至AML阶段
- AML阶段无需重复填写公司名称、注册号、注册国家
- UBO阶段使用KYB+AML的完整企业信息

**显示方式**：
- 在新阶段顶部用绿色信息框展示"已从[前阶段]获取的信息"
- 列出具体字段和值
- 不可编辑（如需修改，需返回原阶段）

## 3. 草稿保存与恢复机制

### 3.1 功能背景
RWA机构认证流程包含**6个阶段、多达20+页面**，用户需要填写大量信息并上传多个文件，完整流程预计需要**25-30分钟**。考虑到：
- 用户可能需要中途离开查找材料
- 网络可能临时中断
- 用户可能需要分多次完成
- 避免用户重复填写造成的挫败感

因此，系统必须提供**自动草稿保存和智能恢复**功能。

### 3.2 自动保存机制

#### 3.2.1 保存时机
**实时自动保存**（无需用户手动操作）：
- 表单字段失焦后立即保存（300ms防抖）
- 文件上传成功后立即保存
- 下拉选择/单选/复选框变更后立即保存
- 点击"下一步"/"返回"按钮时保存
- 页面关闭/刷新前自动保存（beforeunload事件）

**保存提示**：
- 首次自动保存后，右上角显示"已自动保存"提示（3秒后消失）
- 后续保存静默进行，不打扰用户

#### 3.2.2 保存内容
**完整保存的数据**：
1. **当前进度**：
   - 当前所在页面（currentStep）
   - 当前阶段索引（0-3）
   - 各阶段完成状态

2. **KYB阶段数据**：
   - 公司名称、注册号、注册国家
   - 已上传文件列表（文件名、大小、上传时间）
   - 当前上传索引

3. **合规审核阶段数据**：
   - 雇主识别号、注册州、营业地址
   - 城市、州、邮编

4. **UBO阶段数据**：
   - 股东列表（姓名、持股比例、KYC状态）
   - 实际控制人列表（姓名、职位、KYC状态）
   - 是否有UBO人员（hasUBOPersonnel）
   - 无UBO人员原因说明（noPersonnelReason）
   - 无UBO人员证明文件列表（noPersonnelProofs）
   - 声明确认状态（4个复选框）
   - 电子签名数据（签名人、签名时间）

5. **账户操作人阶段数据**：
   - 操作人类型（已认证股东/实际控制人/新增人员）
   - 操作人索引（如选择已认证人员）
   - 操作人姓名和职位（如新增人员）
   - KYC完成状态

6. **专业投资者阶段数据**：
   - 已上传资产证明文件
   - 声明确认状态（agreeToTerms）

7. **机构风险测评数据**：
   - 当前答题进度（currentQuestion）
   - 答案数组（answers，10个问题的答案索引）
   - 总分（totalScore）
   - 测评结果（level、type、description）
   - 声明确认状态（confirmDeclaration）

8. **最终确认数据**：
   - 最终声明确认状态（confirmed）

9. **元数据**：
   - 最后保存时间（用于判断草稿有效期）
   - 设备标识（用于多设备同步提示）
   - 会话ID（用于安全验证）

**不保存的数据**：
- 临时的KYC流程数据（每次KYC完成后清除）
- 密码、Token等敏感信息
- 临时的验证错误提示

#### 3.2.3 保存位置
**优先级1：后端保存**（推荐）
- 保存至数据库草稿表
- 与用户账号绑定
- 支持多设备访问
- 更安全、更可靠

**优先级2：LocalStorage保存**（备选）
- 保存至浏览器本地存储
- Key格式：`rwa_certification_draft_{userId}`
- 仅限当前设备和浏览器
- 容量限制：5-10MB

**混合方案**（最优）：
- 已登录用户：后端保存 + LocalStorage缓存
- 未登录用户：仅LocalStorage保存
- 提交时若检测到LocalStorage有草稿，同步至后端

### 3.3 恢复机制

#### 3.3.1 恢复入口检测
**场景1：用户进入首页**
- 系统检测是否存在未完成的草稿
- 检测条件：
  - 草稿存在
  - 草稿未过期（7天内）
  - 草稿未提交
  - 当前页面状态不是"已完成"

**检测结果**：
- **有草稿**：显示恢复提示弹窗
- **无草稿**：正常显示首页内容

#### 3.3.2 恢复提示弹窗

**弹窗样式**：
- 模态框，半透明遮罩
- 居中显示
- 蓝色边框，白色背景

**弹窗内容**：
```
标题：检测到未完成的认证
图标：🔄（蓝色圆形图标）

内容：
您有一份进行到 [阶段名称] 的认证申请未完成
最后保存时间：YYYY-MM-DD HH:mm

已完成内容：
✓ KYB认证（已完成）
✓ 合规审核（已完成）
⏳ UBO认证（进行中）
  - 已添加 2 位股东
  - 1 位股东待身份认证
○ 账户操作人认证（未开始）
○ 专业投资者认证（未开始）
○ 机构风险测评（未开始）

温馨提示：草稿将在 X 天后过期
```

**操作按钮**：
- **继续填写**（主按钮，蓝色）：恢复草稿，跳转至中断页面
- **重新开始**（次要按钮，灰色）：清空草稿，从头开始
- **关闭**（X按钮）：关闭弹窗，留在首页

#### 3.3.3 恢复流程

**点击"继续填写"后**：
1. 显示加载动画："正在恢复您的进度..."（1-2秒）
2. 从保存位置读取草稿数据
3. 恢复所有表单字段和上传文件
4. 恢复进度条状态
5. 跳转至中断时的具体页面
6. 显示成功提示："已恢复至上次进度"（3秒后消失）

**恢复后的交互**：
- 进度条显示实际进度
- 已填字段显示已填内容（可编辑）
- 已上传文件显示文件名（可重新上传）
- 已完成阶段可点击"返回"查看
- 未完成阶段按正常流程继续

**点击"重新开始"后**：
1. 显示二次确认弹窗："确认重新开始吗？当前进度将被清除。"
2. 确认后清空草稿数据
3. 显示首页初始状态
4. 显示提示："已清除草稿"（3秒后消失）

#### 3.3.4 恢复异常处理

**场景1：草稿数据损坏**
- 检测：JSON解析失败或关键字段缺失
- 提示："草稿数据异常，建议重新开始"
- 仅显示"重新开始"按钮

**场景2：草稿版本不兼容**
- 检测：草稿版本号与当前系统版本不匹配
- 提示："检测到旧版本草稿，部分数据可能丢失，建议重新开始"
- 显示"继续填写（可能有数据丢失）"和"重新开始"按钮

**场景3：后端草稿与本地草稿冲突**
- 检测：后端和LocalStorage同时存在草稿，且保存时间不一致
- 提示："检测到多份草稿，请选择恢复哪一份"
- 显示：
  - 设备A草稿：最后保存时间、进度
  - 本地草稿：最后保存时间、进度
- 按钮："恢复设备A草稿" / "恢复本地草稿" / "重新开始"

### 3.4 草稿有效期管理

#### 3.4.1 有效期规则
- **有效期**：草稿保存后**7天**内有效
- **计算方式**：从最后保存时间开始计算
- **过期处理**：
  - 自动清除过期草稿（每日定时任务）
  - 用户尝试恢复过期草稿时提示："草稿已过期，请重新填写"

#### 3.4.2 过期提醒
- **剩余1天**：用户登录时顶部显示横幅提示
  - "您有一份未完成的RWA认证，将在1天后过期，请尽快完成"
  - "继续填写"按钮
  - 可关闭（X按钮）

### 3.5 多设备同步

#### 3.5.1 后端保存方案
- 草稿绑定用户账号，支持多设备访问
- 设备A保存 → 设备B登录 → 自动同步最新草稿

#### 3.5.2 冲突处理
**场景**：设备A和设备B同时编辑草稿

**处理策略**：
- **最后保存优先**：后保存的数据覆盖先保存的数据
- **冲突提示**：当检测到服务器草稿比本地草稿新时
  - 提示："检测到其他设备有更新的进度，是否同步？"
  - 显示两份草稿的保存时间和进度对比
  - 按钮："同步最新进度" / "保留本地进度" / "重新开始"

### 3.6 草稿清除机制

#### 3.6.1 自动清除时机
- 用户点击"提交完整申请"并成功提交
- 草稿过期（7天后）
- 用户主动选择"重新开始"

#### 3.6.2 手动清除入口
**位置**：用户个人中心 → 认证管理

**显示内容**：
- 草稿状态：进行中
- 当前进度：XX%
- 最后保存：YYYY-MM-DD HH:mm
- 操作按钮：
  - "继续填写"（主按钮）
  - "删除草稿"（次要按钮，红色）

**删除确认**：
- 点击"删除草稿"弹出确认："确认删除草稿吗？此操作不可恢复。"
- 确认后清除草稿数据

### 3.7 用户提示与引导

#### 3.7.1 首次使用提示
**时机**：用户第一次填写表单时

**内容**：
- 浅蓝色提示框（可关闭）
- 图标：💾
- 文字："系统会自动保存您的填写进度，您可以随时离开并继续填写"

#### 3.7.2 保存状态提示
**样式**：右上角浮动提示

**状态1：保存成功**
- 图标：绿色勾选 ✓
- 文字："已自动保存"
- 显示3秒后淡出

**状态2：保存中**
- 图标：旋转加载动画
- 文字："保存中..."
- 保存完成后切换至"已自动保存"

**状态3：保存失败**
- 图标：红色感叹号 ⚠️
- 文字："保存失败，请检查网络"
- 显示"重试"按钮
- 不自动消失

#### 3.7.3 离开提示
**时机**：用户填写过程中尝试关闭页面/浏览器

**提示**：
- 浏览器原生确认框（beforeunload）
- 文字："您有未完成的认证，确认离开吗？系统已自动保存您的进度。"
- 按钮："离开" / "留在此页"

## 4. 第一阶段：KYB认证交互

### 4.1 流程步骤
```
选择国家 → 材料清单 → 逐个上传文件 → 完成
```

### 4.2 核心交互逻辑

#### 4.2.1 国家选择页
**交互要点**：
- 输入框支持搜索过滤
- 点击输入框显示下拉列表
- 选中后输入框显示国家名称
- 国家代码显示在右侧（小字）

**验证规则**：
- 公司名称、注册号、国家为必填
- 所有字段验证通过才能"下一步"

**跳转**：点击"下一步" → 材料清单页

#### 4.2.2 材料清单页
**交互要点**：
- 根据选择的国家动态显示所需材料列表
- 每项材料显示序号、名称、说明
- 已上传的材料显示绿色勾选图标

**信息提示**：
- 蓝色信息框：提示准备材料（PDF/JPG/PNG，最大10MB）

**跳转**：点击"开始上传" → 文件上传页（第一个材料）

#### 4.2.3 文件上传页
**交互要点**：
- 显示当前上传材料名称和说明
- 显示进度："第 X 步，共 Y 步"
- 支持拖拽上传和点击上传
- 未上传状态：虚线边框 + 上传图标
- 已上传状态：绿色边框 + 文件名 + "重新上传"按钮

**上传流程**：
1. 选择文件
2. 自动验证（格式、大小）
3. 验证通过后"下一步"按钮可用
4. 点击"下一步"切换到下一个材料

**循环逻辑**：
- 逐个完成所有材料上传
- 最后一个材料的按钮文字改为"完成上传"
- 点击"完成上传" → 自动跳转至合规审核

## 5. 第二阶段：合规审核交互

### 5.1 流程步骤
```
补充信息表单 → 完成
```

### 5.2 核心交互逻辑

#### 5.2.1 信息展示
**顶部信息框**（绿色）：
- 标题："已从KYB获取的信息"
- 显示：公司名称、注册号、注册国家
- 只读显示，不可编辑

####  5.2.2 表单填写
**必填字段**：
- 雇主识别号（EIN）
- 注册州（下拉选择）
- 营业街道地址
- 城市
- 州（下拉选择）
- 邮编

**页面说明**：
- 页面标题："合规审核"

## 6. 第三阶段：UBO认证交互

### 6.2 核心交互逻辑

#### 6.2.1 添加人员页面

**页面结构**：
- 分为两个独立区域：股东信息和实际控制人信息
- 两个区域互相独立，可同时填写或分别填写

**股东信息部分**：

  1. 点击"+ 添加股东"按钮

**实际控制人信息部分**：
- 标题："实际控制人"
- 说明：实际控制人是指虽然不直接持股25%以上，但通过其他方式实际控制企业的个人

  1. 点击"+ 添加实际控制人"按钮

**无UBO人员场景**：

  5. 点击"提交说明" → 跳转至账户操作人选择页面

- 点击"下一步" → 身份认证页面（有UBO人员时）或直接跳转至账户操作人页面（无UBO人员时）

#### 6.2.2 身份认证页面（KYC集成）

**人员列表展示**：
- 每个人员一个卡片
- 股东卡片：蓝色边框
- 实际控制人卡片：绿色边框
- 显示：姓名、角色标签、持股比例/职位

**认证状态**：
- **待认证**：显示"开始认证"按钮
- **认证中**：显示"认证中..."文字（禁用状态）
- **已完成**：显示绿色勾选图标 + "已完成"文字

**KYC跳转交互**：
1. 点击"开始认证"按钮
2. **跳转至KYC流程**（嵌入式子流程）
3. 该人员状态更新为"认证中"
4. 进入KYC四步流程（详见下方）
5. KYC完成后**自动返回**身份认证页面
6. 该人员状态更新为"已完成"

**提交条件**：
- 所有人员状态都为"已完成"
- "下一步"按钮才可用

#### 6.2.3 KYC子流程交互（嵌入）

**流程概览**：
```
选择证件类型 → 拍摄证件 → 人脸识别 → 确认信息 → 自动返回
```

**第1步：选择证件类型**
- 两个大卡片：身份证、护照
- 点击选中（蓝色边框 + 蓝色背景）
- 显示"✓ 已选择"徽章
- 顶部显示当前认证人员姓名
- 点击"下一步" → 拍摄证件页

**第2步：拍摄证件**
- **身份证模式**：显示两个上传区域（正面+反面）
- **护照模式**：显示一个上传区域（信息页）
- 点击上传区域 → 模拟拍照（500ms后显示已拍摄）
- 已拍摄状态：绿色勾选 + "已拍摄" + "重新拍摄"
- 显示OCR识别进度（2秒加载动画）
- 点击"下一步" → 人脸识别页

**第3步：人脸识别**
- 显示摄像头预览区域（黑色背景）
- 未开始状态：摄像头图标 + "点击下方按钮开始"
- 点击"开始人脸识别"按钮
- 识别中状态：
  - 显示人脸图标
  - 扫描线动画（蓝色，上下移动）
  - 显示识别步骤文字（每0.8秒切换）
  - 步骤：启动摄像头 → 检测人脸 → 活体检测 → 提取特征 → 比对 → 分析相似度
- 识别成功状态：
  - 绿色勾选图标
  - "✓ 人脸识别成功"
  - "相似度：98.5%"
- 识别失败状态：
  - 红色错误图标
  - "✗ 人脸识别失败"
  - "请确保光线充足..."
  - "重新识别"按钮
- 点击"下一步" → 确认信息页

**第4步：确认信息**
- 绿色信息框提示：信息由AI自动识别
- 显示OCR识别的所有字段（可编辑）
- 字段包括：姓名、性别、民族、出生日期、证件号、地址、签发机关、有效期
- 点击"确认并完成认证" → **自动返回UBO身份认证页面**
- 该人员状态自动更新为"已完成"

**取消交互**：
- 任何KYC步骤都可点击"取消"按钮
- 返回UBO身份认证页面
- 该人员状态恢复为"待认证"

#### 6.2.4 声明确认页面

**UBO列表展示及签名**：
- 标题："最终受益所有人列表及签名"

- 每个UBO人员独立卡片显示

**股东卡片样式**：
- 蓝色边框 + 蓝色背景
- 显示：姓名、"股东"角色、持股比例
- UBO标签
- **独立签名区域**：
  - 未签名：虚线边框 + "请签署确认" + "点击签名"按钮
  - 已签名：绿色边框 + 勾选图标 + "已签名" + 签名人姓名 + "重新签名"按钮

**实际控制人卡片样式**：
- 绿色边框 + 绿色背景
- 显示：姓名、职务
- 实际控制人标签
- **独立签名区域**（同股东卡片）

**签名交互流程**：
1. 点击个人卡片的"点击签名"按钮
2. 弹出prompt对话框："请 [姓名] 输入姓名进行电子签名"
3. 输入姓名并确认
4. 自动生成签名数据（姓名+时间戳）
5. 该人员卡片显示已签名状态

**声明复选框**（4个必选）：
1. 确认信息真实、准确、完整
2. 承诺30天内更新变更信息

  - 标题："请完成以下所有步骤才能提交："
  - 列出未完成项（动态显示）
  - 显示签名进度："请所有股东和实际控制人完成签名（已完成 X / Y 人）"
- 所有条件满足后"完成UBO认证"按钮可用

**签名验证逻辑**：

- 签名人数统计：`已签名人数 / 总人数`
- 任何一人未签名，提交按钮保持禁用

**跳转**：点击"完成UBO认证" → 自动跳转至账户操作人认证

## 7. 第四阶段：账户操作人认证交互

### 7.1 功能背景与目标

**功能目的**：
- 为机构账户指定授权的操作人员
- 该操作人的人脸信息将用于后续账户登录验证
- 确保账户安全性和操作可追溯性

**设计理念**：
- 灵活性：可选择已认证的股东/董事，或新增专门的操作人员
- 便捷性：已认证人员无需重复KYC，新增人员需完成KYC
- 安全性：人脸识别确保登录人员身份

### 7.2 流程步骤
```
选择账户操作人（职位） → KYC认证（可选） → 完成
```

### 7.3 核心交互逻辑

#### 7.3.1 选择账户操作人页面

**人员列表展示**：

   - 必填验证

2. **职位下拉选择**：
   - 标签："职位 *"
   - 选项：
     - 财务总监（CFO）

**记录的数据**：
```javascript
operatorForm: {
```

### 7.5 异常处理

#### 7.5.1 无已认证人员
**场景1**：UBO阶段没有任何已完成KYC的人员（但有添加人员）

**处理**：
- 已认证人员列表显示："暂无已认证人员"
- 仅显示"添加新的操作人"选项
- 用户必须新增并完成KYC认证

**场景2**：UBO阶段选择了"没有股东和实际控制人"

**处理**：
- 不显示"从已认证人员中选择"区域
- 仅显示"添加新的操作人"选项
- 用户必须新增操作人并完成KYC认证
- 系统记录该机构无UBO人员的状态

#### 7.5.2 KYC认证失败
**场景**：新增操作人的KYC认证失败（人脸识别失败等）

**处理**：
- 参考UBO阶段的KYC异常处理
- 允许重新认证（无次数限制）
- "取消"按钮返回选择页面

#### 7.5.3 表单验证失败
**场景**：新增操作人时姓名或职位未填写

**处理**：
- 实时显示黄色提示框
- "下一步"按钮保持禁用状态
- 提示具体缺失的字段

## 9. 第六阶段：机构风险测评交互

### 9.1 功能背景与目标

**功能目的**：
- 评估机构的风险承受能力
- 确保机构投资者适配高风险RWA产品
- 符合监管要求的投资者适当性管理

**产品要求**：

- 仅适合**成长型（R4）及以上**的机构投资者参与
- R1-R3级别的机构无法通过认证

### 9.2 流程步骤
```
欢迎页 → 问卷答题（10题） → 声明确认 → 查看结果 → 最终确认
```

### 9.3 核心交互逻辑

#### 9.3.1 风险测评欢迎页（`risk-welcome`）

**页面标题**：
- "第六步：机构风险承受能力测评"
- 副标题："请根据贵机构的真实情况完成以下测评"

**重要提示**（红色信息框）：
- 图标：⚠️

## 12. 异常处理与边界场景

### 12.1 中断恢复
**场景**：用户在任何阶段退出流程

**处理**：
- 草稿自动保存至本地/后台
- 下次进入时显示提示："检测到未完成的认证，是否继续？"
- 点击"继续" → 跳转至中断页面
- 点击"重新开始" → 清空草稿，从首页开始

### 12.2 网络异常
**场景**：文件上传中网络断开

**处理**：
- 显示上传失败提示
- 保留已上传文件列表
- 提供"重试"按钮
- 不清空已填写数据

### 12.3 KYC认证失败
**场景**：人脸识别失败或OCR识别失败

**处理**：
- 显示失败原因
- 提供"重新识别"按钮
- 允许重试（无次数限制）
- 不影响其他人员的认证状态

### 12.4 签名无效
**场景**：用户在签名prompt中点击取消或输入空白

**处理**：
- 显示提示："请输入有效的姓名"（alert）
- 签名区域保持未签名状态
- "完成UBO认证"按钮保持禁用

### 12.5 文件格式错误
**场景**：用户上传不支持的文件格式

**处理**：
- 拒绝上传
- 弹出提示："仅支持PDF/JPG/PNG格式"
- 清空文件选择
- 上传区域恢复初始状态

### 12.6 账户操作人选择异常
**场景**：选择新增操作人但未完成KYC认证就尝试退出

**处理**：
- 保存已选择的操作人类型和基本信息（草稿）
- 下次恢复时提示："账户操作人认证未完成，是否继续？"
- 继续认证或重新选择

### 12.7 风险测评不通过
**场景**：机构风险测评结果为R1、R2或R3级别

**处理**：
- 显示不符合产品要求的提示页面
- 提供"重新测评"选项（允许用户重新答题）
- 提供"退出认证"选项（返回首页，清空草稿）
- 不允许继续提交认证申请

**重新测评规则**：
- 清空所有答案和测评结果
- 返回风险测评欢迎页
- 重新开始答题流程
- 无重新测评次数限制

## 13. 状态管理规范

### 13.1 全局状态
```javascript
{
  currentStep: 'home' | 'kyb-country' | 'kyb-materials' | 'kyb-upload' | 
               'aml-input' | 
               'ubo-personnel' | 'ubo-no-personnel' | 'ubo-verification' | 'ubo-declaration' |
               'operator-select' | 'operator-kyc-complete' |
               'kyc-select-id-type' | 'kyc-capture-id' | 'kyc-face-recognition' | 'kyc-confirm-info' |
               'pi-upload' | 
               'risk-welcome' | 'risk-questionnaire' | 'risk-declaration' | 'risk-result-pass' | 'risk-result-fail' |
               'final-declaration' |
               'final-success'
}
```

### 13.2 阶段状态
每个阶段维护独立的数据对象：
- `kybForm`: KYB阶段数据
- `amlForm`: 合规审核阶段数据
- `uboForm`: UBO阶段数据
  - `shareholders`: 股东列表（每个股东包含`signature`字段）
  - `directors`: 实际控制人列表（每个实际控制人包含`signature`字段）
  - `hasUBOPersonnel`: 是否有UBO人员（布尔值）
  - `noPersonnelReason`: 无UBO人员的原因说明（字符串）
  - `noPersonnelProofs`: 无UBO人员的证明文件列表（数组）
  - `declaration`: 声明确认状态（4个复选框）
- `operatorForm`: 账户操作人数据（包含selectedPersonType、selectedPersonIndex、name、position、kycCompleted）
- `kycForm`: KYC临时数据（用于单个人员认证，支持shareholder、director、operator类型）
- `piForm`: 专业投资者数据
  - `assetProof`: 资产证明文件
  - `declaration.agreeToTerms`: 声明确认
- `riskForm`: 风险测评数据
  - `currentQuestion`: 当前题目索引
  - `answers`: 答案数组（10个问题）
  - `totalScore`: 总分
  - `confirmDeclaration`: 声明确认状态
  - `result`: 测评结果（level、type、description、suitableProducts）
- `finalDeclaration`: 最终声明数据
  - `confirmed`: 最终声明确认状态

### 13.3 状态同步规则
- KYB完成 → 数据传递至合规审核（公司名称、注册号、国家）
- UBO人员KYC完成 → 更新该人员的`kycStatus`为`completed`
- 所有UBO人员KYC完成 → 启用"下一步"按钮
- 选择"没有股东和实际控制人" + 完成说明和上传证明 → 直接跳转至账户操作人认证
- UBO所有人员完成签名 + 声明确认完成 → 启用"完成UBO认证"按钮
- 账户操作人选择 + KYC完成（如需） → 启用"继续下一步"按钮
- 专业投资者声明确认 → 启用"继续风险测评"按钮
- 风险测评所有问题回答完成 → 启用"提交测评"按钮
- 风险测评声明确认 → 启用"查看结果"按钮
- 风险测评通过（R4或R5）→ 显示"继续最终确认"按钮
- 风险测评不通过（R1-R3）→ 显示"重新测评"和"退出认证"按钮
- 最终声明确认 → 启用"提交完整申请"按钮

### 15.2 术语表
- **KYB**: Know Your Business，了解你的企业
- **AML**: Anti-Money Laundering，反洗钱（现已更名为"合规审核"）
- **UBO**: Ultimate Beneficial Owner，最终受益所有人（包括25%以上股东和实际控制人）
- **KYC**: Know Your Customer，了解你的客户
- **RWA**: Real World Assets，真实世界资产
- **OCR**: Optical Character Recognition，光学字符识别
- **账户操作人**: 机构授权的账户管理人员，其人脸信息用于登录验证
- **风险测评**: 评估机构风险承受能力的问卷调查
- **R1-R5**: 风险等级分类，从保守型（R1）到进取型（R5）
